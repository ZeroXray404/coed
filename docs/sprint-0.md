# 4. Gruppens arbetssätt.

## Hur planerar ni att kommunicera inom gruppen?

Kommunikationen kommer att bygga på GitHub Flow. Detta innebär att vårt arbete kommer att följa en specifikt uppsatt process.Den innebär att vi utgår från main som innehåller alla godkända filer i vår applikation. När vi startar en ny uppgift, alltså när en ny task utformad från en userstory ska påbörjas, skapar vi alltid en ny Branch. När en task/uppgift/issue är klar skapar vi en Pull Request (PR), vilket gör att vi också kan automatisera uppdateringen av projekttavlan. PR ska alltid granskas innan den mergas in i main. Detta underlättar för oss att hålla en bra kvalitet i vårt arbete då vi kan validera och hjälpa varandra att hålla en så bra kodstandard som vi kan.

Vi har flera verktyg för kommunikation:
### Discord: 
Används som en hub för kommunikation. 
En stor del av den dagliga kommunikationen sker i en discord-chatt. 
Discord kan också ses som ett bibliotek för dokumentation. 
Vi samlar här länkar till de olika verktygen vi använder såsom Google Docs dokument, förslag på verktyg att använda och Whiteboard. 
Här har vi också en kanal för att dokumentera de beslut som tas inom gruppen. 

### Discord Whiteboard: 
Används för en effektivare dialog då vi lättare kan brainstorma och dela/visa förslag. 

### Google Docs: 
Här skriver vi våra gemensamma inlämningar innan de påbörjas här. 
Verktyget kan också användas för att skriva noteringar under möten.

### Planeringstavla:
Vi använder oss av två olika planeringstavlor med olika syften. 

GitHub Projects: 
Här kommer alla uppgifter som är kopplade till kravspecifikationen/userstories/tasks att läggas till, tilldelas en utvecklare och tidsestimering göras. 

I Trello: 
Här har vi skapat en gemensam arbetsyta och kommer att jobba i en tavla för varje sprint. 
I Trello lägger vi upp planerade möten med Agenda. Alla har möjlighet att lägga till punkter/ kort för saker som man vill lyfta på kommande möte. 
Vi samlar också kort som beskriver vilka inlämningar som ska göras under sprinten, om det är något som behöver läsas eller videor som alla bör titta på. Detta kan var och en sedan ha som en checklista för att påminnas om vad som ska göras. 
Här samlar vi sådant som inte är direkt kopplad till kravspecifikationen och tasks men som behöver utföras/hållas koll på. 

### Google Cal: 
Här lägger vi in alla planerade möten såsom planeringsmöten, dagliga standups och reflektionsmöten. 

## Hur ska ni fördela arbete?
Vi kommer att bryta ner kravspecifikationen i olika user stories. Dessa kommer vi sedan göra om till tasks med acceptance criteria som vi estimerar tiden på och tilldelar till en av gruppmedlemmarna.

## Har ni en plan för om samarbetet inte fungerar som tänkt - har ni ett kontrakt för hur samarbetet ska gå till?
Vi kommer ha daily standup för att se hur det går för alla, asynkron daily standup för de som inte har möjlighet till den planerade daily standup i röstsamtal på Discord.
Vi respekterar varandra och vi antas ta eget ansvar över arbetet. Vi är alla relativt upptagna så mycket tid går ju åt annat såsom jobb, familj, möten, osv. Därför kommunicerar vi istället med varandra om hur det står till, och hjälper varandra om man skulle stå fast någonstans.

## Vilket planeringsverktyg har ni valt och varför?
Vi startade med Trello då det var enkelt att börja jobba i och för att snabbt skapa en gemensam överblick över vad som behöver göras i Sprint 0. Vi vill gärna använda GitHub Projects när vi kommer igång med tasks i Sprint 1 och framåt eftersom en task i GitHub projects kan kopplas till det vi gör i Github. 
Vi tror att båda verktygen kan komplettera varandra med olika syften och planerar därför att jobba i både Trello och Github Projects.
